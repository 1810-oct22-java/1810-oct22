Threads

    What Is Multithreading

  • Multithreading is a type of execution model that allows multiple threads to exist within the context. of a process such that they execute independently but share their process resources.
  • Threading can be useful by allowing the main execution thread to be responsive to user, input, while the additional worker thread can execute long-running that do not need user intervention in the background.

    Thread States

  • New: A thread that has not yet started is in this state.
  • Runnable: A thread executing in the JVM is in this state.
  • Blocked: A thread that is blocked is waiting for a monitor lock is in this state.
  • Waiting: A thread that is waiting indefinitely for another thread to perform a particular action is in this state.
  • Timed Waiting: A thread that is waiting for another thread to perform an action for up to a specified waiting time is in this state.
  • Terminated A thread that has exited is in this state.

    Thread Priorities

  • Thread priority is used to determine when threads switch from running to waiting.
  • A thread with a higher priority will begin running over a lower priority without warning.
  • To set the priority of a thread we use the setPriority() function. This is a method of the Thread class.
  • Usually when setting thread priority we use integers, however in place of integers MIN_PRIORITY, NORM_PRIORITY, and MAX_PRIORITY can be substituted.

    Synchronization

  • When using threads these objects can behave in a asynchronous manner. One thread can attempt to read data that another thread is writing. This leads to inconsistency.
  • In order to counter this there needs to be a way to stop more than one thread from acccessing a resource at a time. This is achieved by using Synchronization.
  • To implement this Java has a synchronous method. When using this method all other threads wait until the thread that is using a resource within this synchronous method is finished with the process.
