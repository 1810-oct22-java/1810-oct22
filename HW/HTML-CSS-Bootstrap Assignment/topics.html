Collection/Collections

Collection

  • A collection represents a group of objects, knowns as its elements. Some Collections allow duplicate elements and others do not. Some are ordered and others unordered
  • It provides implementations of more specific subinterfaces like Set and List
  • Some implementations of the Collection interface don't allow for null elements, and some have restrictions on the types of their elements.

Collections

  • The Collections class consists exclusively of static methods that operate on or return collections.
  • List
    • An ordered collection
    • You can dictate where to insert an element into the list
    • Allow for duplicate elements
  • Set
    • An unordered collection
    • Cannot contain duplicates
