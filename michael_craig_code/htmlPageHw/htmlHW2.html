Joins

Relational databases are structured such that each table holds both uniquely important data, and data which maintains a relationship with one or more other tables. When one wants to interact with pieces of data from two or more tables, a join can come in handy.
There are many different types of joins to fulfill every use case. The most basic and commonly used is the INNER JOIN. As shown in the diagram, inner joins evaluate the data from two tables and return the matching or overlapping values. In other words, rows that match are accepted, and rows that don't are rejected.

Next are OUTER JOINS. There are three different types: full outer, left outer, and right outer. As seen in the left-most image, the left outer join will return all the rows from the left table, as well as those rows which match the table on the right. If there are no columns on the right that match the left, null values are displayed.
As dipicted by the image in the center, the full outer join returns all values from both tables where the specified conditions are met, and displays null in instances where there is no match.
Finally, as seen in the image on the right, the right outer join will function in the same way that a left outer join would, except the table on the right is displayed along with the matching values.
